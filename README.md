🦾 Multik-RAG: Автономный ИИ-агент с динамическим контекстом и защитой

Продвинутый Telegram-агент на базе Llama 3.1 (Ollama), реализующий архитектуру RAG (Retrieval-Augmented Generation). Проект оптимизирован для работы в закрытых контурах (Air-gapped) и на ограниченных аппаратных ресурсах (NVIDIA 1660 Ti 6GB VRAM).

🚀 Ключевые возможности
Multimodal RAG Engine: Интеллектуальный поиск по базе знаний ФНС (PDF/Markdown) с использованием эмбеддингов multilingual-e5-small. 🏎️💨
Стриминг ответов (SSE): Реализация потоковой передачи токенов в реальном времени для минимизации задержек (TTFT).
Динамический кэш медиа: Автоматическое сопоставление текстовых ответов с визуальными ассетами (скриншотами ИРМ) через кэширование file_id Telegram.
Безопасность (Hard Filter): Многоуровневая система защиты от промпт-инъекций и попыток взлома системной роли (50+ векторов атаки).
Fine-tuned Inference: Поддержка кастомных адаптеров (LoRA) через Unsloth для специализации модели под предметную область.

🛠 Технологический стек
Ядро: Python 3.10+, Aiogram 3.x (Asyncio)
AI Frameworks: LlamaIndex, Ollama (Llama 3.1), Unsloth (для тренировки)
Векторная база: Local Vector Store (Disk-persistent)
DevOps: Docker, Docker Compose (GPU Passthrough)
Оптимизация: Dynamic Path Resolution (авто-поиск снапшотов моделей), Resource Limits (60W/80W GPU Power Management)

🛡 Архитектура безопасности
Система реализует паттерн "Sanitizer Layer". Каждый входящий запрос проходит через регулярную фильтрацию и валидацию Pydantic-моделей. При обнаружении попыток сброса контекста (Jailbreak), агент переходит в режим блокировки, предотвращая доступ к системным промптам.
