
🦾 Multik-RAG: Суверенный Автономный ИИ-Контур ФНС РФ
Изолированный локальный ИИ-агент на базе отечественной модели YandexGPT-5-Lite-8B (Реестр российского ПО Минцифры), реализующий архитектуру Enterprise RAG (Retrieval-Augmented Generation) [146-ФЗ]. Комплекс спроектирован для работы в абсолютно закрытых сетевых периметрах (Air-gapped) без доступа к интернету и оптимизирован под жесткие аппаратные лимиты (NVIDIA GPU 6GB VRAM) [146-ФЗ].

🏎️ Ключевые возможности контура
100% Air-gapped деплой: Сборка и запуск происходят строго из локального кэша Docker-образов, гарантируя нулевую зависимость от внешних репозиториев.
Суверенное ядро LLM: Модель YandexGPT-5-Lite-8B-Instruct в кванте Q4_K_M целиком помещается в VRAM и обеспечивает юридическую чистоту ответов по нормативной базе ФНС [146-ФЗ].
Гоночный пайплайн COMPACT: Переработка ядра RerankedEngine исключила медленные древовидные суммаризаторы, обеспечив сквозной проход по чанкам документов за один такт.
Аддитивный Entity Boost: Математика ранжирования переведена с RRF-умножения на сложение скоров с весовым коэффициентом (+ bonus * 0.1) через глобальный граф сущностей (graph_global.json), исключая галлюцинации [146-ФЗ].
Аудит токенов: Интегрирован локальный неубиваемый счетчик токенов на базе кодировщика cl100k_base для контроля заполнения контекстного окна [146-ФЗ].
Стриминг (SSE): Потоковая выдача ответов со скоростью 15.0 токенов/сек, убирающая психологическое ожидание у пользователя.
Защита от сикофантии (Policy Control): 100% устойчивость к коррупционным провокациям и стресс-тестам. Система вырезает «воду» и возвращает бескомпромиссный токен БАЗА_ПУСТА при отсутствии жесткой нормы права в контексте [146-ФЗ].

📊 Карта утилизации ресурсов (6 ГБ VRAM Лимит)
Благодаря разделению обязанностей и оптимизации драйверов, система работает на пределе физических возможностей железа без рисков OOM (Out of Memory):
Монополия на GPU: Контейнер FastAPI (multik_api) полностью лишен доступа к GPU; эмбеддинги и реранкер RoSBERTa принудительно переведены на CPU. Видеокарта на 100% отдана Ollama.
CUDA Сжатие: Флаг OLLAMA_KV_CACHE_TYPE=q8_0 сжимает KV-кэш истории диалога из Float16 в Int8 на уровне драйвера, укладывая окно в 6144 токенов в VRAM [146-ФЗ].
Хирургическая точность памяти: Процесс Ollama занимает ровно 5590 MiB из 6144 MiB, оставляя системе Ubuntu стабильный зазор в 550 МБ.
Тайминги (End-to-End Latency): Полный цикл RAG-ответа занимает ~34 сек (~16 сек тяжелый реранкинг RoSBERTa на CPU + ~18 сек полная генерация развернутого ответа на GPU).

🛠 Технологический стек
Инфраструктура: Docker, Docker Compose (GPU Passthrough, CUDA Toolkit).
Служебные флаги ядра Ollama: OLLAMA_NUM_PARALLEL=1 (блокировка параллельных сессий против CUDA OOM), OLLAMA_MAX_QUEUE=128 (честная системная очередь запросов), OLLAMA_KEEP_ALIVE=-1 (перманентное удержание модели в VRAM для мгновенного отклика).
Бэкенд: Python 3.10+, Aiogram 3.x (Asyncio), FastAPI.
ИИ-Пайплайн: LlamaIndex (Core), Ollama Runner, RoSBERTa Reranker.
